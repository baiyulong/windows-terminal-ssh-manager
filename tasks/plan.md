# Plan: wt-ssh-manager — Windows Terminal SSH Manager

## Overview

A modern SSH server manager built on **Windows Terminal JSON Fragment Extensions**.
Each saved server becomes a native Windows Terminal profile (with its own icon, title,
and color scheme). Credentials are encrypted with **Windows DPAPI** so only the current
OS user can decrypt them. Connections are established via a `launcher.py` script that
runs inside the Terminal tab and drives an interactive Paramiko session.

## Architecture

```
┌───────────────────────────────────────────────┐
│  Windows Terminal (host)                       │
│   ├─ Profile: "🖥 prod-web"  ──────────────┐  │
│   ├─ Profile: "🖥 dev-db"    ──────────────┤  │  (Fragment JSON)
│   └─ Profile: "🖥 staging"   ──────────────┘  │
└───────────────────────────────────────────────┘
           ↑ auto-generated by fragment.py
           
~/.wt-ssh-manager/
  ├─ config.json          ← server list (passwords encrypted with DPAPI)
  └─ hosts/               ← per-server icon overrides (future)

%LOCALAPPDATA%\Microsoft\Windows Terminal\Fragments\wt-ssh-manager\
  └─ profiles.json        ← Fragment injected into Windows Terminal

wt-ssh-manager/
  ├─ ssh_manager/
  │   ├─ __init__.py
  │   ├─ crypto.py        ← Windows DPAPI encrypt/decrypt
  │   ├─ config.py        ← Server data model + persistence
  │   ├─ fragment.py      ← WT Fragment JSON generator
  │   ├─ launcher.py      ← Interactive SSH session (runs IN Terminal tab)
  │   └─ cli.py           ← click-based CLI: add/list/remove/edit/sync/test
  ├─ main.py              ← Entry point  (`python main.py <cmd>`)
  ├─ requirements.txt
  └─ install.bat          ← One-click setup
```

## Technology Stack

| Concern | Choice | Why |
|---------|--------|-----|
| CLI framework | `click` | composable commands, auto `--help` |
| Terminal UI | `rich` | tables, prompts, spinners, colors |
| SSH | `paramiko` | pure-Python SSH2, no external binary needed |
| Encryption | Windows DPAPI via `ctypes` | ties secrets to current OS user, no master password |
| Config format | JSON (`~/.wt-ssh-manager/config.json`) | human-readable, easy to backup |

## Server Data Model

每台服务器存储独立的账号密码，密码通过 Windows DPAPI 加密（只有当前 OS 用户可解密）。

```json
{
  "servers": [
    {
      "id": "prod-web",
      "name": "prod-web",
      "host": "192.168.1.100",
      "port": 22,
      "username": "root",
      "encrypted_password": "<dpapi-base64>",
      "description": "Production web server",
      "tags": ["prod", "linux"],
      "color": "one-half-dark"
    }
  ]
}
```

**密码存储流程：**
1. `add` 时通过 `rich.prompt.Prompt.ask(password=True)` 安全输入密码
2. 立即用 DPAPI 加密后写入 `config.json`，明文只在内存里短暂存在
3. `launcher.py` 启动时解密密码，交给 Paramiko，连接成功后密码从内存中清除
4. Windows Terminal 中打开 Profile → **自动连接，无需手动输密码**

## Windows Terminal Fragment Output

每台服务器的 `commandline` 指向 `launcher.py`，打开 Tab 时自动用存储的账号密码连接：

```json
{
  "profiles": [
    {
      "guid": "{<uuid5>}",
      "name": "🖥  prod-web",
      "commandline": "python \"C:\\path\\to\\launcher.py\" prod-web",
      "tabTitle": "prod-web",
      "colorScheme": "One Half Dark",
      "startingDirectory": "%USERPROFILE%",
      "icon": "🖥"
    }
  ]
}
```

**连接流程（用户体验）：**
1. 在 Windows Terminal 菜单中点击服务器名
2. 新 Tab 打开，`launcher.py` 自动解密密码，调用 Paramiko 连接
3. 连接成功后直接进入交互式 Shell，**全程无需手动输入密码**
4. 支持 ANSI 颜色、方向键、Ctrl+C 等所有终端操作

## CLI Commands

| Command | Description |
|---------|-------------|
| `ssh-manager add` | Interactive wizard: name, host, port, user, auth, password/key |
| `ssh-manager list` | Rich table of all servers |
| `ssh-manager remove <name>` | Delete server + regenerate fragment |
| `ssh-manager edit <name>` | Edit any field interactively |
| `ssh-manager sync` | Regenerate Windows Terminal Fragment JSON |
| `ssh-manager test <name>` | Test connection (banner + latency) |
| `ssh-manager connect <name>` | Open interactive session in current terminal |

## Phases

### Phase 1 — Foundation (Tasks 1-3)
Bootstrap → Crypto → Config.  No UI yet.  Verifiable via Python REPL.

### Phase 2 — Terminal Integration (Task 4)
Fragment generator. Profiles appear in Windows Terminal after save.

Launcher script. `python launcher.py <name>` opens a live SSH shell.

### Phase 4 — Full CLI (Task 6)
All `click` commands wired together.

### Phase 5 — Polish (Task 7)
install.bat, error messages, edge-case handling.
